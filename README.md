# 🛠️ دليل النشر والتثبيت لمنصة Netlify | WAM Services

مرحباً بك في دليل النشر الخاص بتطبيق **دليل مقدمي الخدمات الفنية والمهنية في اليمن**.
تم تصميم وتجهيز هذا المشروع ليعمل بشكل فوري وكامل كـ **Single Page Application (SPA)** تعتمد على React وVite، وهو مهيأ تماماً للنشر على منصة **Netlify** العالمية.

---

## 📸 ملفات النشر الأساسية المتضمنة في مجلد `dist`
عند بناء المشروع للموقع، يتم توليد المجلد `dist` والذي يحتوي تلقائياً على الملفات التالية (كما هو موضح في متطلبات الصورة المرفقة):
1. **`_redirects`**: ملف التوجيه الخاص بـ Netlify لمنع حدوث خطأ 404 عند تحديث الصفحة (يحتوي على `/* /index.html 200`).
2. **`assets/`**: ملفات الأكواد والأنماط المترجمة والمضغوطة (JS & CSS).
3. **`favicon.svg`**: الأيقونة الرسمية للتطبيق.
4. **`index.html`**: ملف الدخول الرئيسي للموقع.
5. **`opengraph.jpg`**: صورة المعاينة الفاخرة عند مشاركة رابط الموقع على وسائل التواصل الاجتماعي.
6. **`robots.txt`**: ملف توجيه محركات البحث لضمان أرشفة ممتازة.
7. **`sw.js`**: ملف الـ Service Worker لدعم التشغيل دون إنترنت (Offline Cache).

## 🚀 خطوات النشر على Netlify (طريقتان)

### الطريقة الأولى: السحب والإفلات المباشر (سهلة وسريعة جداً)
1. قم بإنشاء أو تسجيل الدخول إلى حسابك في [Netlify](https://app.netlify.com/).
2. اذهب إلى صفحة **Sites** في لوحة التحكم.
3. قم بالتمرير لأسفل الصفحة حتى تجد صندوق السحب والإفلات المكتوب عليه **"Want to deploy a new site without connecting to Git? Drag and drop your site folder here"**.
4. قم بسحب وإفلات مجلد `dist` المولد (أو الملف المضغوط بعد فك ضغطه) داخل الصندوق.
5. سيتم نشر موقعك فوراً وتوليد رابط مباشر مجاني! 🎉

---

### الطريقة الثانية: النشر التلقائي عبر GitHub (الموصى بها للتحديثات التلقائية)
1. قم برفع كود المشروع بالكامل إلى مستودع (Repository) خاص بك على **GitHub**.
2. اذهب إلى لوحة تحكم **Netlify** واضغط على **Add new site** ثم اختر **Import an existing project**.
3. اختر **GitHub** وقم بتفويض Netlify للوصول إلى المستودع الخاص بك.
4. حدد المستودع الخاص بالتطبيق وقم بضبط إعدادات البناء كالتالي:
   - **Branch to deploy**: `main` (أو الفرع الرئيسي لديك)
   - **Build command**: `npm run build`
   - **Publish directory**: `dist`
5. اضغط على **Deploy site** ليبدأ النظام ببناء ونشر الموقع تلقائياً مع كل تحديث ترسله لـ GitHub!

## 🔑 إعداد المتغيرات البيئية (Environment Variables)

لضمان عمل جميع الميزات السحابية ومساعد الذكاء الاصطناعي بشكل صحيح، يجب إضافتها في لوحة تحكم Netlify تحت تبويب **Site settings > Environment variables** (في حال استخدام الطريقة الثانية) أو تضمينها في إعدادات التطبيق:

| اسم المتغير البيئي | الوصف | الحالة |
| :--- | :--- | :--- |
| `VITE_FIREBASE_API_KEY` | مفتاح Firebase API الخاص بقاعدة البيانات السحابية | اختياري (توجد قاعدة محلية تلقائية) |
| `VITE_FIREBASE_AUTH_DOMAIN` | نطاق المصادقة لـ Firebase Auth | اختياري |
| `VITE_FIREBASE_PROJECT_ID` | معرف مشروع Firebase | اختياري |
| `VITE_FIREBASE_STORAGE_BUCKET` | مساحة التخزين لـ Firebase | اختياري |
| `VITE_FIREBASE_MESSAGING_SENDER_ID` | معرف الإرسال لـ FCM (الإشعارات) | اختياري |
| `VITE_FIREBASE_APP_ID` | معرف تطبيق Firebase الخاص بك | اختياري |

> 💡 **ملاحظة هامة:** يحتوي التطبيق على آلية **Local Fallback** ذكية جداً وقاعدة بيانات محلية تعمل بالكامل في حال عدم توفر المتغيرات السحابية أو انقطاع الاتصال بالإنترنت، لضمان استمرارية الخدمة بنسبة 100%!

## 🎨 مميزات وتحديثات الإصدار الأخير
* **حركات انسيابية فاخرة**: دمج **Framer Motion** لمنح التنقل بين التبويبات حركة انسيابية وتجربة مستخدم لا مثيل لها.
* **توافق كامل مع Netlify**: يحتوي على ملف `_redirects` معد مسبقاً لضمان عمل الروابط والتبويبات بشكل صحيح ودون مشاكل إعادة التحميل.
* **أداء فائق السرعة**: تم تقليل حجم الملفات الناتجة وضغط الأصول البرمجية لأقصى درجة لضمان تحميل فوري حتى في أضعف شبكات الإنترنت في اليمن.
